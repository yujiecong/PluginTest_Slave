# 03 - 渲染管线设计

## 1. 概述

PyManimPlugin 的渲染管线负责两件事：
1. **实时预览**——在编辑器 Viewport 中实时显示场景
2. **离线输出**——将指定帧渲染为 PNG 图片文件

视频合成不在 UE 侧完成，由 Python 使用 ffmpeg 将图片序列打包为视频。

## 3. 实时预览模式

### 工作原理

- `UPyManimScene::Initialize()` 时创建一个 `APyManimSceneActor`
- Scene Actor 包含一个 `UCameraComponent`，用作场景的主相机
- 用户在编辑器中看到的 Viewport 就是该相机的视角
- Mobject 的 Actor 直接生成在世界中，对相机可见
- 用户可以在编辑器 Viewport 中自由拖拽旋转缩放，不影响 Python 脚本的执行

### 编辑器工作流

```
1. 打开 UE5 编辑器
2. Window → Developer Tools → Python Console
3. 在 Python 控制台粘贴或导入脚本
4. 脚本执行时，在编辑器 Viewport 中实时看到结果
5. 可以在 Viewport 中拖拽视角，辅助调试
```

## 4. 离线渲染模式

### 架构

```
UPyManimScene::RenderFrame(FilePath)
         │
         ▼
┌─────────────────┐
│ SceneCapture2D   │  ← 一个隐藏的 2D 场景捕获器
│ (独立于编辑器)   │     使用与 Scene Camera 相同的位置/旋转/FOV
└────────┬────────┘
         │ 捕获当前帧
         ▼
┌────────────────┐
│ RenderTarget2D  │  ← 内存中的纹理缓冲区
│ (Size = 用户指定)│     分辨率由 Initialize(Width, Height) 决定
└────────┬───────┘
         │ ExportRenderTarget
         ▼
┌────────────────┐
│ ImageWriteQueue │  ← UE 异步图片写出队列
│ (后台线程)      │     不阻塞主线程
└────────┬───────┘
         │
         ▼
   [PNG 文件]
   output/frame_0001.png
```

### 设计理由：为什么不用 MovieRenderPipeline？

| 方案 | 优点 | 缺点 |
|------|------|------|
| **MovieRenderPipeline** | 完整的电影级输出、抗锯齿、后处理 | 初始化重量级、配置复杂、依赖队列系统、不适合单帧输出 |
| **SceneCapture2D + RenderTarget** | 轻量、任意帧捕获、即时输出 | 需手动控制抗锯齿、无音轨 |
| **PyManim 选型** | ✅ 选 SceneCapture2D | 只需出图，ffmpeg 合视频更灵活 |

### CaptureFrameSequence 内部逻辑

```cpp
void UPyManimRenderer::CaptureFrameSequence(
    const FString& OutputDir, int32 StartFrame, int32 EndFrame,
    TArray<UPyManimAnimation*> Animations)
{
    float TotalDuration = 0.0f;
    for (auto& Anim : Animations)
    {
        TotalDuration += Anim->GetDuration();
    }
    float FPS = (EndFrame - StartFrame) / TotalDuration;
    float FrameInterval = 1.0f / FPS;

    for (int32 Frame = StartFrame; Frame <= EndFrame; ++Frame)
    {
        float Time = Frame * FrameInterval;
        // 对所有动画 Tick
        for (auto& Anim : Animations)
        {
            float AnimProgress = (Time - AnimStartTime) / Anim->GetDuration();
            Anim->Tick(FrameInterval, FMath::Clamp(AnimProgress, 0.0f, 1.0f));
        }
        // 捕获
        FString Path = FString::Printf(TEXT("%s/frame_%04d.png"), *OutputDir, Frame);
        CaptureFrame(Path);
    }
}
```

## 6. 视频合成（Python 侧）

```python
import subprocess
import os

def frames_to_video(frames_dir: str, output_path: str, fps: int = 30):
    """Python 侧调用 ffmpeg 将图片序列合成为视频"""
    cmd = [
        "ffmpeg",
        "-framerate", str(fps),
        "-i", os.path.join(frames_dir, "frame_%04d.png"),
        "-c:v", "libx264",
        "-pix_fmt", "yuv420p",
        "-crf", "18",
        "-y",
        output_path
    ]
    subprocess.run(cmd, check=True)
```

### 也可以嵌入到 Scene API 中

```python
# Python 封装：一键渲染动画到视频
def render_scene_to_video(scene, output_path: str, duration: float, fps: int = 30):
    import tempfile, os, subprocess

    with tempfile.TemporaryDirectory() as tmpdir:
        scene.render_frames(tmpdir, duration, fps)

        subprocess.run([
            "ffmpeg", "-framerate", str(fps),
            "-i", os.path.join(tmpdir, "frame_%04d.png"),
            "-c:v", "libx264", "-pix_fmt", "yuv420p",
            "-crf", "18", "-y", output_path
        ], check=True)

    print(f"Video saved to: {output_path}")

# 使用
render_scene_to_video(scene, "D:/output/my_scene.mp4", duration=10.0, fps=30)
```

## 7. 性能与质量考虑

| 项目 | 默认值 | 说明 |
|------|--------|------|
| 分辨率 | 1920×1080 | 可调，通过 `Scene::Initialize(W, H)` |
| 输出格式 | PNG (RGBA) | 支持透明背景，便于后期合成 |
| 抗锯齿 | 由 RenderTarget 设置 | 建议 MSAA 4x ~ 8x |
| 后处理 | 继承 Editor 设置 | Bloom、Tonemapping 等自动生效 |
| 异步写出 | ImageWriteQueue | 不阻塞渲染线程 |

## 8. CLI 模式下渲染

CLI 模式下没有 Editor Viewport，完全依赖 SceneCapture2D：

```
UE5Editor-Cmd.exe PluginTest.uproject -run=PyManimCommandlet -Script=myscene.py

内部流程:
  1. Commandlet 启动
  2. 运行 python myscene.py
  3. Python 调用 scene.render_frames(...)
  4. SceneCapture2D 逐帧抓取 → PNG 输出
  5. Python 调用 ffmpeg 合成视频
  6. UE 退出
```

## 9. 渲染管线扩展计划

| 阶段 | 内容 |
|------|------|
| Phase 1 (MVP) | 基础 SceneCapture2D 单帧/序列输出 |
| Phase 2 | 自定义后处理材质链（如：卡通渲染、线框模式） |
| Phase 3 | 高分辨率离线渲染（4K/8K，分块渲染合图） |
| Phase 4 | 多相机同时录制（可切换视角的视频） |
| Phase 5 | 透明背景 + Alpha 通道输出（便于在其他工具中合成） |
