# 03 - 渲染管线设计

> **文档版本**: v2.0 (2026-05-07 更新)
> **实现状态**: ✅ 已完成（Phase 3）
> **核心组件**: UUEMotionRenderer (基于 MovieRenderPipeline)

## 1. 概述 ✅

UEMotionPlugin 的渲染管线负责两件事：
1. **实时预览**——在编辑器 Viewport 中实时显示场景
2. **离线输出**——通过 **MovieRenderPipeline (MRP)** 将指定帧序列渲染为高质量 PNG 图片文件

视频合成不在 UE 侧完成，由 Python 使用 ffmpeg 将图片序列打包为视频。

### 当前实现状态

| 功能 | 状态 | 说明 |
|------|:----:|------|
| 实时预览 | ✅ 已实现 | Editor Viewport 实时渲染 |
| MRP 离线渲染 | ✅ 已实现 | 高质量 PNG 序列输出 |
| 单帧截图 | ✅ 已实现 | RenderSingleFrame() |
| 序列帧渲染 | ✅ 已实现 | RenderSequence() |
| ffmpeg 视频合成 | ✅ 已实现 | Python 端自动调用 |
| 抗锯齿控制 | ✅ 已实现 | TemporalAA / MSAA |
| 多格式输出 | ✅ 已实现 | PNG / EXR / JPG |
| 后处理链 | ⚠️ 部分 | 继承 Editor PostProcess |

## 3. 实时预览模式

### 工作原理

- `UEMotionScene::Initialize()` 时创建一个 `AUEMotionSceneActor`
- Scene Actor 包含一个 `UCameraComponent`，用作场景的主相机
- 用户在编辑器中看到的 Viewport 就是该相机的视角
- Mobject 的 Actor 直接生成在世界中，对相机可见
- 用户可以在编辑器 Viewport 中自由拖拽旋转缩放，不影响 Python 脚本的执行

### 编辑器工作流

```
1. 打开 UE5 编辑器
2. Window → Developer Tools → Python Console
3. 在 Python 控制台粘贴或导入脚本
4. 脚本执行时，在编辑器 Viewport 中实时看到结果
5. 可以在 Viewport 中拖拽视角，辅助调试
```

## 4. 离线渲染模式 — MovieRenderPipeline

### 为什么选 MRP？

| 方案 | 优点 | 缺点 |
|------|------|------|
| **SceneCapture2D + RenderTarget** | 轻量、任意帧捕获 | 抗锯齿有限、无后处理链、手动管理复杂 |
| **MovieRenderPipeline** | 电影级质量、内置 AA/后处理/多格式、UE 官方维护 | 初始化略重、需配置队列 |
| **UEMotion 选型** | ✅ 选 **MovieRenderPipeline** | 离线渲染追求质量 > 轻量 |

## 6. Build.cs 依赖更新 ✅

```cpp
void UEMotionRenderer::Initialize(UWorld* InWorld, int32 Width, int32 Height)
{
    WorldContext = InWorld;
    OutputWidth = Width;
    OutputHeight = Height;

    // 1. 创建或获取 MRP 队列
    PipelineQueue = NewObject<UMoviePipelineQueue>(this);
    PipelineQueue->AddToRoot();

    // 2. 创建主配置
    MasterConfig = NewObject<UMoviePipelineMasterConfig>(this);
    MasterConfig->AddToRoot();

    // 3. 设置分辨率
    FIntPoint Resolution(Width, Height);
    MasterConfig->SetResolution(Resolution);

    // 4. 设置默认帧率
    MasterConfig->SetFrameRate(FFrameRate(30, 1));

    // 5. 设置抗锯齿为 TemporalAA（高质量）
    MasterConfig->SetAntiAliasingMethod(EAntiAliasingMethod::AAM_TemporalAA);

    // 6. 创建输出设置
    OutputSetting = NewObject<UMoviePipelineOutputSetting>(this);
    OutputSetting->AddToRoot();

    bInitialized = true;
}
```

### RenderSequence 内部实现

```cpp
void UEMotionRenderer::RenderSequence(
    const FString& OutputDirectory, float Duration, float FPS)
{
    if (!bInitialized || !BoundCamera.IsValid()) return;

    // 1. 计算帧数
    int32 TotalFrames = FMath::CeilToInt(Duration * FPS);

    // 2. 更新 MasterConfig
    MasterConfig->SetFrameRate(FFrameRate(FPS, 1));
    MasterConfig->SetResolution(FIntPoint(OutputWidth, OutputHeight));

    // 3. 配置输出路径和格式
    OutputSetting->OutputDirectory.Path = OutputDirectory;
    OutputSetting->OutputFormat = EImageFormat::PNG;
    OutputSetting->FileNameFormat = TEXT("frame.{frame_number}");

    // 4. 创建 Job 并关联配置
    UMoviePipelineExecutorJob* Job = PipelineQueue->AllocateNewJob();
    Job->SetConfiguration(MasterConfig);
    Job->MapPath = FSoftObjectPath(WorldContext.Get()->GetFName());

    // 5. 关联输出设置
    Job->GetResolvedSettings().Add(OutputSetting);

    // 6. 绑定相机
    if (auto* Cam = BoundCamera.Get())
    {
        Job->SetCameraBinding(Cam);
    }

    // 7. 设置帧范围
    Job->SetSequenceNumberRange(TRange<FFrameNumber>(
        FFrameNumber(0),
        FFrameNumber(TotalFrames - 1)));

    // 8. 注册每帧回调（用于驱动动画 Tick）
    // 通过 MoviePipeline 的 PreRenderTick 回调推进动画

    // 9. 开始渲染
    PipelineQueue->StartQueuedJobs();
}

// 每帧回调：驱动动画进度
void UEMotionRenderer::OnPreRenderTick(float DeltaTime)
{
    if (!SceneInstance) return;
    SceneInstance->Tick(DeltaTime);  // 推进所有活跃动画
}
```

### 单帧渲染（快速预览）

对于单帧截图需求（调试用），仍可使用轻量的方案：

```cpp
void UEMotionRenderer::RenderSingleFrame(const FString& FilePath)
{
    if (!bInitialized || !BoundCamera.IsValid()) return;

    // 方案 A：使用 UKismetRenderingLibrary（轻量单帧）
    UWorld* World = WorldContext.Get();
    if (!World) return;

    auto* Camera = BoundCamera.Get();

    // 用 SceneCapture2D 快速截取一帧（仅用于单帧预览）
    // 序列帧批量渲染统一走 MRP
    FIntPoint Size(OutputWidth, OutputHeight);
    UTextureRenderTarget2D* RT = UKismetRenderingLibrary::CreateRenderTarget2D(
        World, Size.X, Size.Y, RTF_RGBA16f, FLinearColor::Black);

    // 同步相机位置并抓取
    UKismetRenderingLibrary::ExportRenderTarget(
        World, RT, FilePath, FImageExportParams());

    RT->ConditionalBeginDestroy();
}
```

## 7. 视频合成（Python 侧）

```python
import subprocess
import os

def frames_to_video(frames_dir: str, output_path: str, fps: int = 30):
    """调用 ffmpeg 将 MRP 输出的图片序列合成为视频"""
    cmd = [
        "ffmpeg",
        "-framerate", str(fps),
        "-i", os.path.join(frames_dir, "frame.*.png"),
        "-c:v", "libx264",
        "-pix_fmt", "yuv420p",
        "-crf", "18",
        "-preset", "medium",
        "-y",
        output_path
    ]
    subprocess.run(cmd, check=True)

# 一键渲染动画到视频
def render_scene_to_video(scene, output_path: str, duration: float, fps: int = 30):
    import tempfile

    with tempfile.TemporaryDirectory() as tmpdir:
        scene.render_frames(tmpdir, duration, fps)
        frames_to_video(tmpdir, output_path, fps)

    print(f"Video saved to: {output_path}")

# 使用
render_scene_to_video(scene, "D:/output/my_scene.mp4", duration=10.0, fps=30)
```

## 8. 性能与质量考虑

| 项目 | 默认值 | 说明 |
|------|--------|------|
| 分辨率 | 1920×1080 | 可调，通过 `Initialize(W, H)` 或 `SetResolution()` |
| 帧率 | 30fps | 可调 |
| 抗锯齿 | TemporalAA | 可选: None / TemporalAA / MSAA×2 / MSAA×4 |
| 输出格式 | PNG (RGBA) | 支持 EXR（16bit 浮点）、JPG |
| 后处理 | 继承 Editor PostProcess | Bloom / Tonemapping / ColorGrading 自动生效 |
| 透明背景 | EXR 格式支持 Alpha | 用于后期合成 |
| 多线程 | MRP 内置并行 | 利用多核加速渲染 |

### MRP vs SceneCapture2D 对比

| 维度 | MovieRenderPipeline | SceneCapture2D |
|------|---------------------|---------------|
| 抗锯齿 | ✅ TemporalAA / MSAA | ⚠️ 仅 MSAA |
| 后处理 | ✅ 完整 PP 链 | ❌ 需手动复现 |
| 输出格式 | ✅ PNG / EXR / JPG | ✅ PNG |
| Alpha 通道 | ✅ EXR RGBA | ⚠️ 需特殊配置 |
| 性能 | 较慢（逐帧全管线） | 较快（直接抓取） |
| 适用场景 | 最终输出 | 调试预览 |

**策略**：
- **批量离线渲染** → 走 MRP（高质量）
- **单帧快速截图** → 走 UKismetRenderingLibrary（轻量）

## 9. CLI 模式下渲染

CLI 模式下没有 Editor Viewport，完全依赖 MRP：

```
UE5Editor-Cmd.exe PluginTest.uproject -run=UEMotionCommandlet -Script=myscene.py

内部流程:
  1. Commandlet 启动
  2. 运行 python myscene.py
  3. Python 调用 scene.render_frames(...)
  4. MRP 在后台逐帧渲染（无需视口）
  5. 输出 PNG 序列帧到磁盘
  6. Python 调用 ffmpeg 合成视频
  7. UE 退出
```
