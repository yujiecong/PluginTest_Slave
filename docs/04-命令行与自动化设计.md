# 04 - 命令行与自动化设计

## 1. 概述

PyManimPlugin 支持两种运行模式：

| 模式 | 场景 | 启动方式 |
|------|------|----------|
| **交互模式** | 编辑器内调试、手动开发 Python 脚本 | 打开 UE5Editor，Python Console 粘贴脚本 |
| **CLI 模式** | 自动化/批量渲染、CI/CD 流水线 | `UE5Editor-Cmd.exe ... -run=PyManimCommandlet` |

两种模式共享同一套 C++ API，同一个 Python 脚本可以无缝切换。

## 2. UPyManimCommandlet 设计

```cpp
UCLASS()
class UPyManimCommandlet : public UCommandlet
{
    GENERATED_BODY()

public:
    UPyManimCommandlet();

    // ========== 入口 ==========
    virtual int32 Main(const FString& Params) override;

private:
    // ========== 参数解析 ==========
    bool ParseParameter(const FString& Params, const FString& Key, FString& OutValue);

    // ========== 执行 Python 脚本 ==========
    bool ExecutePythonScript(const FString& ScriptPath);

    // ========== 初始化编辑器世界 ==========
    bool InitializeEditorWorld();
};
```

### Main 函数内部流程

```cpp
int32 UPyManimCommandlet::Main(const FString& Params)
{
    // 1. 解析参数
    FString ScriptPath;
    if (!ParseParameter(Params, TEXT("-Script="), ScriptPath))
    {
        UE_LOG(LogPyManim, Error, TEXT("Missing -Script= parameter"));
        return 1;
    }

    // 2. 初始化编辑器环境（轻量世界）
    if (!InitializeEditorWorld())
    {
        UE_LOG(LogPyManim, Error, TEXT("Failed to initialize editor world"));
        return 1;
    }

    // 3. 执行 Python 脚本
    if (!ExecutePythonScript(ScriptPath))
    {
        UE_LOG(LogPyManim, Error, TEXT("Python script execution failed"));
        return 1;
    }

    // 4. 正常退出
    UE_LOG(LogPyManim, Log, TEXT("PyManimCommandlet completed successfully"));
    return 0;
}
```

## 3. 启动命令

### 基本用法

```bash
# Windows
UE5Editor-Cmd.exe "C:/Users/.../PluginTest/PluginTest.uproject" -run=PyManimCommandlet -Script=myscene.py

# 也可以使用绝对路径
UE5Editor-Cmd.exe "../PluginTest.uproject" -run=PyManimCommandlet -Script="D:/scripts/myscene.py"
```

### 扩展参数（未来规划）

```bash
UE5Editor-Cmd.exe PluginTest.uproject -run=PyManimCommandlet \
    -Script=myscene.py \
    -OutputDir="D:/output/" \
    -Resolution=3840x2160 \
    -FPS=60 \
    -Timeout=300 \
    -Log=verbose
```

| 参数 | 说明 |
|------|------|
| `-Script=` | 必填，Python 脚本路径 |
| `-OutputDir=` | 输出目录（覆盖脚本中的设置） |
| `-Resolution=` | 渲染分辨率 (WxH) |
| `-FPS=` | 帧率 |
| `-Timeout=` | 超时时间（秒），防止脚本死循环 |
| `-Log=` | 日志级别 |

## 4. Python 脚本规范

用户在 `Scripts/` 目录下编写 Python 脚本时，需遵循以下约定：

### 最小入口

```python
# myscene.py
import unreal

def main():
    scene = unreal.PyManimScene()
    scene.initialize(width=1920, height=1080)

    # ... 你的场景、动画代码 ...

    scene.render_frames("D:/output/frames/", duration=5.0, fps=30.0)

if __name__ == "__main__":
    main()
```

### 推荐的脚本结构

```python
"""
my_geometry_demo.py
PyManim 示例：几何变换动画演示
"""

import unreal


# ==========================================
# 场景配置
# ==========================================
SCENE_WIDTH = 1920
SCENE_HEIGHT = 1080
OUTPUT_DIR = "D:/output/geometry_demo/"
FPS = 30
DURATION = 8.0


# ==========================================
# 构建场景
# ==========================================
def build_scene(scene: unreal.PyManimScene):
    """创建场景中的所有物体"""

    # 光照
    scene.add_directional_light([0, -1, -1], unreal.LinearColor(1, 1, 1, 1), 8.0)
    scene.set_ambient_light(unreal.LinearColor(0.2, 0.2, 0.25, 1))

    # 相机
    camera = scene.get_camera()
    camera.set_position(-400, -500, 300)
    camera.look_at_position([0, 0, 0])

    # 参考网格和坐标轴
    scene.create_grid(1200.0, 12)

    return scene


# ==========================================
# 定义动画序列
# ==========================================
def animate(scene: unreal.PyManimScene):
    """编排动画"""

    # 创建球体并动画
    sphere = scene.create_sphere(60.0)
    sphere.set_location([-200, 0, 50])
    sphere.set_color(unreal.LinearColor(1, 0.3, 0.3, 1))

    move = sphere.create_move_animation()
    move.set_target([200, 0, 50])
    move.set_duration(3.0)
    scene.play(move)

    # 创建旋转的立方体
    cube = scene.create_cube(80.0)
    cube.set_location([0, 0, 250])
    cube.set_color(unreal.LinearColor(0.3, 0.5, 1, 1))

    rotate = cube.create_rotate_animation()
    rotate.set_rotation_angle(720)
    rotate.set_duration(6.0)
    scene.play(rotate)


# ==========================================
# 主函数
# ==========================================
def main():
    scene = unreal.PyManimScene()
    scene.initialize(width=SCENE_WIDTH, height=SCENE_HEIGHT)

    build_scene(scene)
    animate(scene)

    # 离线渲染
    scene.render_frames(OUTPUT_DIR, duration=DURATION, fps=FPS)
    print(f"Done! Frames saved to: {OUTPUT_DIR}")


if __name__ == "__main__":
    main()
```

## 5. Commandlet 执行 Python 的方式

```cpp
bool UPyManimCommandlet::ExecutePythonScript(const FString& ScriptPath)
{
    // UE5 内置 Python 插件提供的 API
    // 方式 1：使用 PythonScriptPlugin 的接口
    IPythonScriptPlugin* PythonPlugin = IPythonScriptPlugin::Get();
    if (!PythonPlugin)
    {
        return false;
    }

    // 构建执行命令
    FString Command = FString::Printf(
        TEXT("exec(open(r'%s', encoding='utf-8').read())"),
        *ScriptPath
    );

    return PythonPlugin->ExecPythonCommand(*Command);
}
```

## 6. 自动化流水线示意

### 本地批量渲染

```bash
#!/bin/bash
# render_all.sh — 批量渲染所有 Python 脚本

SCRIPTS_DIR="./Scripts"
OUTPUT_DIR="./Output"
UE_CMD="/path/to/UE5Editor-Cmd.exe"
UPROJECT="./PluginTest.uproject"

for script in "$SCRIPTS_DIR"/*.py; do
    name=$(basename "$script" .py)
    echo "Rendering: $name"

    "$UE_CMD" "$UPROJECT" \
        -run=PyManimCommandlet \
        -Script="$script" \
        -OutputDir="$OUTPUT_DIR/$name/"

    echo "Done: $name"
    echo "---"
done

echo "All scenes rendered!"
```

### Python 封装（子进程调用）

```python
# render_batch.py
import subprocess
import os
from pathlib import Path

UE_CMD = "C:/Program Files/Epic Games/UE_5.7/Engine/Binaries/Win64/UnrealEditor-Cmd.exe"
UPROJECT = "C:/Users/42458/Documents/Unreal Projects/PluginTest/PluginTest.uproject"

def render_script(script_path: str, output_dir: str):
    cmd = [
        UE_CMD,
        UPROJECT,
        "-run=PyManimCommandlet",
        f"-Script={script_path}",
        f"-OutputDir={output_dir}",
    ]
    result = subprocess.run(cmd, capture_output=True, text=True)
    if result.returncode != 0:
        print(f"Error: {result.stderr}")
    return result.returncode == 0

# 批量渲染
scripts_dir = Path("./Scripts")
for py_file in scripts_dir.glob("*.py"):
    print(f"Processing: {py_file.name}")
    render_script(str(py_file), f"./Output/{py_file.stem}/")
```

## 8. 重要约束

| 约束 | 说明 |
|------|------|
| **编辑器进程** | CLI 模式每次启动一个全新的编辑器进程，脚本执行完就退出 |
| **状态不持久** | 每次 CLI 执行都是新的世界，不能跨脚本共享状态 |
| **启动时间** | UE5 编辑器冷启动约 10-30 秒，批量渲染需考虑这个成本 |
| **Python 版本** | 使用 UE5 内置的 Python（默认 3.11+），与系统 Python 隔离 |
| **路径格式** | Windows 上建议用 `/` 或原始字符串 `r""` 避免转义问题 |

## 9. 对比：编辑器内 vs CLI

| 维度 | 编辑器内 (交互模式) | CLI (Commandlet) |
|------|---------------------|-------------------|
| 启动速度 | 慢（完整编辑器 UI） | 较慢（无 UI，但仍是完整引擎） |
| 调试体验 | ✅ 优秀，Viewport 实时看 + 断点 | ❌ 只能靠日志 |
| 自动化 | ❌ 需要手动操作 | ✅ 脚本驱动、可 CI/CD |
| Python 控制台 | ✅ 可用 | ❌ 不可用 |
| 性能 | 编辑器有 UI 开销 | 略好（无 UI 渲染） |
| 适用场景 | 开发、调试、快速预览 | 批量渲染、最终输出 |
