# 01 - 架构总览与模块设计

## 1. 项目定位

**PyManimPlugin** 是一个 UE5 **Editor-Only 插件**，核心理念：

> UE5 作为纯渲染平台，所有逻辑由 Python 驱动。

类似 [Manim](https://github.com/3b1b/manim) 的思想——用代码描述场景和动画，引擎负责渲染输出。但不同于 Manim 自己画 OpenGL，PyManimPlugin 利用 UE5 的完整渲染管线（PBR、光照、后处理），让用户通过简洁的 Python API 创作 3D 可视化内容。

## 2. 设计目标

| 目标 | 说明 |
|------|------|
| **渲染平台化** | UE5 插件只负责"接收指令 → 渲染画面"，不包含业务逻辑 |
| **蓝图 C++ API** | 所有核心功能用 C++ 实现，标记 `BlueprintCallable` / `BlueprintType`，UE 内置 Python 桥接自动暴露给 `unreal` 模块 |
| **双模式运行** | 编辑器内交互调试 + CLI 无头渲染，同一套 API |
| **对象模型** | 模仿 Manim 的 Scene / Camera / Mobject 设计，Python 端代码风格一致 |
| **离线 + 实时** | 既支持逐帧渲染为图片序列（配合 ffmpeg 合成视频），也支持编辑器视口实时预览 |

## 5. Build.cs 模块依赖

```csharp
// Plugins/PyManimPlugin/Source/PyManimPlugin/PyManimPlugin.Build.cs
using UnrealBuildTool;

public class PyManimPlugin : ModuleRules
{
    public PyManimPlugin(ReadOnlyTargetRules Target) : base(Target)
    {
        PCHUsage = PCHUsageMode.UseExplicitOrSharedPCHs;

        PublicDependencyModuleNames.AddRange(new string[] {
            "Core",
            "CoreUObject",
            "Engine",
            "ProceduralMeshComponent",    // 程序化几何体生成
            "RenderCore",
            "RHI"
        });

        PrivateDependencyModuleNames.AddRange(new string[] {
            "Slate",
            "SlateCore",
            "UnrealEd",                    // Editor Only
            "PythonScriptPlugin",          // Python 桥接
            "ImageWriteQueue",             // 异步图片写出
            "Networking"                   // 未来扩展：socket 通信
        });
    }
}
```

## 8. 架构关键决策

| 决策 | 理由 |
|------|------|
| **Editor Only** (非 Runtime) | 只在编辑器环境使用，不需要打包到游戏 |
| **UObject + BlueprintType** | UE 内置 Python 桥接自动暴露属性/方法，无需 pybind11 |
| **不使用 MovieRenderPipeline** | 过于复杂。直接 RenderTarget + ImageWriteQueue 截图更轻量 |
| **视频合成放 Python 侧** | 用 ffmpeg/subprocess 合图，UE 只负责出帧，职责清晰 |
| **CLI 用 Commandlet** | UE 原生的命令行入口，`UE5Editor-Cmd.exe -run=...` |

## 9. 项目前置条件

使用本插件前，项目需要：

1. **启用 Python 脚本插件**
   - 在 `PluginTest.uproject` 中启用 `PythonScriptPlugin`
   - 或通过编辑器菜单：Edit → Plugins → Scripting → Python Script Plugin

2. **项目 uproject 追加配置**：
```json
// PluginTest.uproject 中追加
"Plugins": [
    { "Name": "PythonScriptPlugin", "Enabled": true },
    { "Name": "PyManimPlugin", "Enabled": true }
]
```

3. **Python 环境**：UE5 内置 Python 3.11+，无需额外安装。
