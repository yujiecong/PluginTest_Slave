## 1. 项目定位

**UEMotionPlugin** 是一个 UE5 **Editor-Only 插件**，核心理念：

> UE5 作为纯渲染平台，所有逻辑由 Python 驱动。

类似 [Manim](https://github.com/3b1b/manim) 的思想——用代码描述场景和动画，引擎负责渲染输出。但不同于 Manim 自己画 OpenGL，UEMotionPlugin 利用 UE5 的完整渲染管线（PBR、光照、后处理），让用户通过简洁的 Python API 创作 3D 可视化内容。

### 当前状态（v0.1.0）

✅ **已完成功能**：
- 完整的对象模型（Scene / Camera / Mobject）
- 6 种基础形状（Sphere, Cube, Cylinder, Cone, Plane, Torus）
- 8 种动画类型（Move, Rotate, Scale, Fade, Color, Wait, Group + Easing）
- MovieRenderPipeline 离线渲染（高质量 PNG 序列）
- ffmpeg 视频合成
- Python 封装层（更简洁的 API）
- 16 个自动化测试
- 5 个示例脚本

## 2. 设计目标

| 目标 | 说明 |
|------|------|
| **渲染平台化** | UE5 插件只负责"接收指令 → 渲染画面"，不包含业务逻辑 |
| **蓝图 C++ API** | 所有核心功能用 C++ 实现，标记 `BlueprintCallable` / `BlueprintType`，UE 内置 Python 桥接自动暴露给 `unreal` 模块 |
| **双模式运行** | **当前**：编辑器内交互调试（Python Console）；**后续**：CLI 无头渲染（Phase 5+ 独立模块） |
| **对象模型** | 模仿 Manim 的 Scene / Camera / Mobject 设计，Python 端代码风格一致 |
| **离线 + 实时** | 既支持逐帧渲染为图片序列（配合 ffmpeg 合成视频），也支持编辑器视口实时预览 |

using UnrealBuildTool;

public class UEMotionPlugin : ModuleRules
{
    public UEMotionPlugin(ReadOnlyTargetRules Target) : base(Target)
    {
        PCHUsage = PCHUsageMode.UseExplicitOrSharedPCHs;

        PublicDependencyModuleNames.AddRange(new string[] {
            "Core",
            "CoreUObject",
            "Engine",
            "ProceduralMeshComponent",    // 程序化几何体生成

            "RHI"

        PrivateDependencyModuleNames.AddRange(new string[] {

    }
}

## 8. 架构关键决策（已验证）

| 决策 | 实际实现 | 理由 |
|------|---------|------|
| **Editor Only** (非 Runtime) | ✅ 已实现 | 只在编辑器环境使用，不需要打包到游戏 |
| **UObject + BlueprintType** | ✅ 已实现 | UE 内置 Python 桥接自动暴露属性/方法，无需 pybind11 |
| **StaticMeshComponent**（非 ProceduralMesh） | ✅ 已实现 | 使用 UE 内置基础形状（Sphere/Cube等），更稳定 |
| **不使用 SceneCapture2D 做批量渲染** | ✅ 已实现 | 批量离线渲染走 MovieRenderPipeline 获得电影级质量 |
| **视频合成放 Python 侧** | ✅ 已实现 | 用 ffmpeg/subprocess 合图，UE 只负责出帧 |
| **Python 封装层** | ✅ 已实现 | `Scripts/uemotion/` 提供 Pythonic API，支持颜色字符串、链式调用等 |
| **CLI 用 Commandlet（Phase 5+）** | ❌ 未实现 | 后续实现：当前聚焦编辑器内使用 |

## 11. 快速开始指南

### 方式一：使用原生 API（unreal 模块）

```python
import unreal

# 创建场景
scene = unreal.UEMotionScene()
scene.initialize(width=1920, height=1080)

# 添加光照
scene.add_directional_light([0, -1, -1], unreal.LinearColor(1, 1, 1, 1), 10)

# 设置相机
camera = scene.get_camera()
camera.set_position(-300, -400, 200)
camera.look_at_position([0, 0, 0])

# 创建物体
sphere = scene.create_sphere(50)
sphere.set_color(unreal.LinearColor(1, 0.2, 0.2, 1))
sphere.set_location([0, 0, 50])

# 渲染输出
scene.render_frames("D:/output/frames/", duration=2.0, fps=30)
```

### 方式二：使用 Python 封装层（推荐 ✅）

```python
from uemotion import Scene

# 一键创建场景
s = Scene(1920, 1080)

# 简洁的API - 支持颜色字符串、链式调用
ball = s.sphere(50, color="red", location=[0, 0, 50])
cube = s.cube(60, color="blue", location=[150, 0, 0])

# 动画 + 渲染一体化
s.play(ball.move_to([200, 0, 50], duration=2))
s.render("D:/output/demo.mp4", duration=5)
```

## 12. 已知限制与后续计划

### 当前版本（v0.1.0）限制

| 限制 | 说明 | 计划 |
|------|------|------|
| 仅6种基础形状 | 缺少 Arrow/Line/Grid/Axes/Text 等 | Phase 4+ |
| 无CLI模式 | 只能在编辑器内使用 | Phase 5+ |
| 无粒子系统 | 不支持特效 | Phase 4+ |
| 单相机渲染 | 不支持多机位同时录制 | Phase 4+ |

### 后续开发优先级

1. **高级形状扩展** - Arrow, Line, Circle, Grid, CoordinateAxes, Text
2. **后处理链** - 卡通渲染、线框模式、Bloom控制
3. **性能优化** - 批量动画、LOD系统
4. **CLI模式** - Commandlet支持批量渲染和CI/CD集成
