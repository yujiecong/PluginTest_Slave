## 2. Phase 1 — 基础搭建（MVP）

### 目标
插件能跑起来，能够从 Python 创建一个球体并看到它。

### 任务清单

| # | 任务 | 产出 |
|---|------|------|
| 1.1 | 创建插件目录结构 | `Plugins/PyManimPlugin/` 完整骨架 |
| 1.2 | 编写 `.uplugin` 描述文件 | 插件注册到项目 |
| 1.3 | 编写 `Build.cs` | 模块依赖正确链接 |
| 1.4 | 实现模块入口 `PyManimPluginModule` | 插件在编辑器中加载 |
| 1.5 | 实现 `UPyManimScene` 最小版本 | `Initialize()` / `CreateSphere()` |
| 1.6 | 实现 `UPyManimMobject` 最小版本 | `SetLocation()` / `SetColor()` |
| 1.7 | 实现 `UPyManimBlueprintLibrary` | `CreateBoxMesh` 等几何体生成 |
| 1.8 | 实现 `APyManimSceneActor` | Scene 在世界的物理载体 |
| 1.9 | 启用 `PythonScriptPlugin` | 确保项目配置正确 |
| 1.10 | 编写 `Scripts/hello_sphere.py` | 验证：Python → C++ → 视口可见球体 |

### 验证标准

```python
# 在 UE 编辑器 Python Console 中执行
import unreal
scene = unreal.PyManimScene()
scene.initialize()
sphere = scene.create_sphere(100)
sphere.set_color(unreal.LinearColor(1, 0, 0, 1))
# → 编辑器 Viewport 中看到一个红色球体
```

### 验证标准

```python
import unreal
scene = unreal.PyManimScene()
scene.initialize(1920, 1080)
camera = scene.get_camera()
camera.set_position(-300, -400, 200)
camera.look_at_position([0, 0, 0])

cube = scene.create_cube(80)

move = cube.create_move_animation()
move.set_target([200, 0, 0])
move.set_duration(2)
move.set_easing("ease_in_out")

scene.play(move)
scene.render_frames("D:/output/test/", duration=2.0, fps=30)
# → 输出 60 帧 PNG，立方体平滑地从原点移动到 (200, 0, 0)
```

## 6. 关键风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| UE Python 桥接不完全支持某些类型 | 中 | 中 | 用基础类型（FVector/String/float）传参，避免复杂模板 |
| ProceduralMeshComponent 性能不足 | 低 | 中 | Mesh 生成放后台线程，Phase 4 考虑 StaticMesh + ISMC |
| UE 版本升级破坏 API | 中 | 高 | 锁定 UE 5.7 为主要开发版本，关注 Epic 升级说明 |
| Python 侧错误难以调试 | 中 | 中 | 编辑器交互模式优先开发（当前方案），CLI 后续再加 |
| MRP 渲染配置复杂度高 | 中 | 中 | 提供预设配置模板（1080p/4K），简化常用场景 |
